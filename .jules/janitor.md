## 2026-05-07 - [Deprecated Provider Cleanup] **Clutter:** Found src/shared/config/schemas.ts still using deprecated @google/genai types and metadata.json being unused. **Learning:** Deleting unused schema files tied to deprecated providers significantly reduces dependency surface area. **Action:** Always cross-reference AI provider deprecations with their associated JSON/Type schemas.
## 2025-05-14 - [Component Export Shadowing] **Clutter:** `src/shared/ui/cards/GenericCard.tsx` exported components (`Card`, `CardTitle`, etc.) that shadowed atomic `card.tsx` exports in `index.tsx`. **Learning:** Wildcard exports from multiple files lead to silent shadowing where named exports take precedence, causing ambiguity in consumers like `CookbookPage.tsx`. **Action:** Use specific prefixes for specialized components (e.g., `MediaCard`) to distinguish them from atomic UI primitives and resolve shadowing in barrel files.
## 2025-05-22 - [Barrel Export Inconsistency] **Clutter:** `GlobalFAB` component was missing from the `shared/ui` barrel export, causing deep imports in page components. **Learning:** Inconsistent barrel exports lead to "import sprawl" and break the abstraction of the shared UI layer. **Action:** Ensure all reusable UI primitives and layout components are exported via `src/shared/ui/index.tsx`.
## 2025-06-01 - [Redundant View Alias Cleanup] **Clutter:** Found `'home'` view being used as a redundant alias for `'cookbook'` in `App.tsx` and `View` type. **Learning:** Redundant view aliases create ambiguity in routing and state management logic. **Action:** Consolidate multiple aliases for the same component into a single, descriptive view name to reduce structural entropy.
